# Atlas

A self-hosted Leaflet map for travel places and GPX trails. Single-file static frontend, optional Python API for in-browser editing. No build step, no Docker.

![Unified map view](docs/screenshots/map.png)

**Docs:** [Configuration](docs/configure.md) · [Security](docs/security.md) · [Contributing](CONTRIBUTING.md)

## Features

- One world map with clustered place pins and GPX trail polylines.
- Filter panel: per-category place toggles, visited / want chips, completed / planned trail chips.
- Browse views: places grouped by category or country, trails grouped by region.
- Trail detail view with elevation profile and route stats.
- Sign in to add, edit, or delete places and trails from the browser. Server strips PII from uploaded GPX (timestamps, author, creator).
- Seven themes (Catppuccin, Dracula, Gruvbox, Nord, Rosé Pine, Solarized, Tokyo Night), each with system / light / dark mode.
- Per-deployment toggle to run as places-only or trails-only; per-browser settings for default tile layer, distance units (metric / imperial), marker clustering, remembering the last map view, pin style (ring / pin / dot / flag), marker size, trail line thickness, and map tile filter.
- Operator can edit category display labels from the Settings UI.
- Works read-only without the API; the static site loads `places.json` and `routes.json` directly.

## Install

You need Python 3.9+ for the API. The static site is just files; serve them with whatever you already use. The walk-through below is the recommended setup: nginx in front, the Python API behind socket-activated systemd, TLS at the proxy.

### 1. Files

Lay the install out at `/srv/atlas`:

```
/srv/atlas/
  index.html
  favicon.svg
  site-config.json          # copy of site-config.example.json
  places.json               # your place data (gitignored upstream)
  gpx/                      # your GPX files (gitignored upstream)
  routes.json               # generated by gpx-manifest.sh
  gpx-manifest.sh
  tools/
    api.py
    config.json             # copy of tools/config.example.json
```

### 2. API config

Edit `tools/config.json`. The minimum:

```json
{
  "bind": "127.0.0.1:8091",
  "db_path": "/srv/atlas/tools/atlas.db",
  "data_dir": "/srv/atlas",
  "manifest_cmd": "/srv/atlas/gpx-manifest.sh",
  "secure_cookies": true,
  "idle_exit_seconds": 300
}
```

To skip the in-browser registration step, also set `initial_user` and `initial_password`. See [docs/configure.md](docs/configure.md) for the full field reference.

### 3. systemd

Atlas ships systemd units for socket activation (zero idle RAM):

```sh
sudo cp deploy/atlas-api.socket /etc/systemd/system/
sudo cp deploy/atlas-api.service /etc/systemd/system/
sudo useradd --system --home /srv/atlas --shell /usr/sbin/nologin atlas
sudo chown -R atlas:atlas /srv/atlas
sudo systemctl daemon-reload
sudo systemctl enable --now atlas-api.socket
```

How it works:
- `atlas-api.socket` holds the port at no cost.
- The first request triggers `atlas-api.service` to start.
- After `idle_exit_seconds` of no activity, the Python process exits. The socket stays held; the next request wakes it again.

### 4. nginx

Copy `deploy/nginx.example.conf` to your nginx config directory. Replace `atlas.example.com`, the cert paths, and `/srv/atlas`, then reload:

```sh
sudo nginx -t && sudo systemctl reload nginx
```

The example config serves static files directly, proxies `/api/*` to the Python process, rate-limits `/api/login` at the proxy layer, and denies access to `tools/` and `deploy/`.

### 5. First sign-in

Open the site. Register the first account; registration auto-closes after that. From the menu, "Settings" lets you toggle visible features, pick a default tile layer, switch units, customize appearance (theme, mode, pin style, marker size, trail thickness, tile filter), change your password, manage active sessions, and (as operator) edit category labels or re-open registration to invite someone.

If you set `require_setup_token: true` in the API config, the first registration also needs the one-time token printed to the API log on startup. Recommended when deploying to the open internet. See [docs/security.md](docs/security.md).

### macOS (launchd)

Same shape, but use `deploy/atlas-api.plist` (template; replace `/Users/YOU/atlas`). launchd does not have systemd-style socket activation, so set `idle_exit_seconds: 0` so the process stays up.

## Read-only deploy

If you don't need editing, skip the API entirely. Drop `index.html`, `favicon.svg`, `site-config.json`, `places.json`, `gpx/`, and `routes.json` on any static host. Atlas loads the data files at runtime; the "Sign in" button will be a dead end.
